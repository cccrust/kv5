## 功能規劃

### 1. 交易支援（Transaction）

Redis 交易確保一批命令的原子性執行。

#### 1.1 新增命令

| 命令 | 說明 | RESP 格式 |
|------|------|-----------|
| `MULTI` | 開始交易，後續命令加入佇列 | SimpleString: "OK" |
| `EXEC` | 執行佇列中的所有命令 | Array |
| `DISCARD` | 清除佇列，結束交易 | SimpleString: "OK" |
| `WATCH <key> [key ...]` | 監控鍵，樂觀鎖 | SimpleString: "OK" |
| `UNWATCH` | 取消所有監控 | SimpleString: "OK" |

#### 1.2 行為規格

1. **MULTI** 後輸入的命令不立即執行，加入佇列
2. **EXEC** 執行佇列所有命令，若被 WATCH 的鍵有變化則全部失敗（回傳空 Array）
3. **DISCARD** 清空佇列並退出交易模式
4. **WATCH** 在 MULTI 前呼叫，樂觀鎖機制
5. 交易中的命令失敗不影響其他命令（無 atomic rollback）

#### 1.3 架構設計

```rust
struct TransactionState {
    queued_commands: Vec<RespValue>,
    in_transaction: bool,
    watched_keys: Vec<String>,
}

struct ClientState {
    tx: TransactionState,
}
```

#### 1.4 實作重點

1. **Client 狀態追蹤**: 每個 client 需要追蹤是否在 MULTI 模式
2. **WATCH 機制**: 記錄 client 監控的鍵，EXEC 前檢查是否被修改
3. **佇列**: 用 Vec 儲存交易中的命令
4. **錯誤處理**: EXEC 時若無命令則返回空 Array

### 2. 效能監控

#### 2.1 INFO 命令

```

# Clients

# Memory

# Stats

total_commands_processed:5000

```

#### 2.2 CLIENT LIST

返回所有客戶端連線資訊

#### 2.3 SLOWLOG

- 記錄執行時間超過閾值的命令
- 最大 128 筆，LRU 淘汰

#### 2.4 ECHO 增強

- 保持現有功能

## API 變更

### 新增命令彙總

| 命令 | 類別 | 說明 |
|------|------|------|
| MULTI | Transaction | 開始交易 |
| EXEC | Transaction | 執行交易 |
| DISCARD | Transaction | 放棄交易 |
| WATCH | Transaction | 監控鍵 |
| UNWATCH | Transaction | 取消監控 |
| INFO | Server | 伺服器資訊 |
| CLIENT LIST | Server | 客戶端列表 |
| CLIENT KILL | Server | 斷開客戶端 |

## 風險評估

1. **交易效能**: WATCH 的樂觀鎖在高競爭場景可能有效能問題
2. **記憶體**: SLOWLOG 需要控制記憶體使用
3. **CLIENT KILL 安全**: 需要驗證客戶端身份
