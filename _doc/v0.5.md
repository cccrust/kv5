## 發展目標

v0.5 主要完成 WATCH 樂觀鎖驗證、連線追蹤、SHUTDOWN，以及其他 Redis 相容性改進。

## 功能規劃

### 1. WATCH 樂觀鎖實現

Redis WATCH 使用樂觀鎖機制，在 EXEC 時檢查被監控的鍵是否有變化。

#### 1.1 實現方式

```rust
// Store 需要維護被監控鍵的版本號/修改計數
struct Store {
    // 現有 fields...
    key_version: DashMap<String, u64>,  // 每個鍵的修改版本
}
```

#### 1.2 行為

1. **WATCH key1 key2**: 記錄客戶端監控的鍵
2. **SET/GET/DEL/...**: 修改鍵時增加 key_version
3. **EXEC**:
   - 檢查所有被監控鍵的 version 是否改變
   - 若改變：返回 Null Array (nil)
   - 若未改變：執行佇列命令

### 2. CLIENT LIST 連線追蹤

#### 2.1 ClientInfo 結構

```rust
struct ClientInfo {
    id: String,          // 客戶端唯一 ID
    addr: String,        // IP:PORT
    db: u8,              // 當前 DB
    cmd: String,         // 最近執行的命令
    idle: u64,           // 空閒時間(秒)
}
```

#### 2.2 全域客戶端追蹤

```rust
// 在 server 或 db module 中
static CLIENTS: DashMap<String, ClientInfo>
```

### 3. SHUTDOWN 命令

```
SHUTDOWN [NOSAVE|SAVE]
```

- **SHUTDOWN SAVE**: 儲存後關閉（預設）
- **SHUTDOWN NOSAVE**: 不儲存直接關閉

### 4. 其他 Server 命令

| 命令 | 說明 |
|------|------|
| `DBSIZE` | 返回 key 數量 |
| `FLUSHALL` | 清除所有 DB |
| `FLUSHDB` | 清除當前 DB |
| `LASTSAVE` | 上次 SAVE 時間戳 |
| `BGSAVE` | 後台儲存 |
| `CONFIG GET` | 取得配置 |
| `CONFIG SET` | 設定配置 |
| `TIME` | 伺服器時間 |

### 5. DEBUG 命令（可選）

- `DEBUG OBJECT key`: 取得 key 的除錯資訊
- `DEBUG SEGFAULT`: 觸發段錯誤（測試用）

## API 變更

### 修改命令

| 命令 | 類別 | 說明 |
|------|------|------|
| EXEC | Transaction | 增加 WATCH 驗證 |
| WATCH | Transaction | 保持相容 |
| CLIENT LIST | Server | 返回實際連線列表 |

### 新增命令

| 命令 | 類別 | 說明 |
|------|------|------|
| SHUTDOWN | Server | 關閉伺服器 |
| DBSIZE | Server | key 數量 |
| FLUSHALL | Server | 清除所有 DB |
| FLUSHDB | Server | 清除當前 DB |
| LASTSAVE | Server | 上次 SAVE 時間 |
| TIME | Server | 伺服器時間 |

## 風險評估

1. **WATCH 效能**: 每次鍵修改都要更新 version，可能有效能影響
2. **CLIENT LIST 安全**: 需確保只能看自己的資訊
3. **SHUTDOWN 安全性**: 需驗證管理員權限
