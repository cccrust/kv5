## 發展目標

v0.6 主要新增 Bit 操作命令，這是 Redis 中強大的位元運算功能。

## 功能規劃

### 1. Bit 操作命令

Redis Bit 操作允許對字串值進行位元級別的讀寫和運算。

#### 1.1 新增命令

| 命令 | 說明 | RESP 格式 |
|------|------|-----------|
| `GETBIT <key> <offset>` | 取得指定位的值（0 或 1） | Integer |
| `SETBIT <key> <offset> <value>` | 設定指定位的值 | Integer（返回原值） |
| `BITCOUNT <key> [start end]` | 計算設定位的數量 | Integer |
| `BITOP <op> <destkey> <key> [key ...]` | 位元運算（AND/OR/XOR/NOT） | Integer |
| `BITPOS <key> <bit> [start] [end]` | 找到第一個指定位的位置 | Integer |

#### 1.2 行為規格

1. **offset**: 從 0 開始，可以超過字串長度，自動擴展
2. **負 offset**: 從字串末尾計算（-1 = 最後一位）
3. **BITCOUNT start/end**: 支援位元組範圍，負值從末尾計算
4. **BITOP NOT**: 只有一個源鍵
5. **BITCOUNT**: 預設計算整個字串

#### 1.3 內部實現

```rust
// 使用 Vec<u8> 儲存位元組陣列
struct BitString(Vec<u8>);

// 取得位
fn get_bit(bytes: &[u8], offset: usize) -> u8 {
    let byte_idx = offset / 8;
    let bit_idx = 7 - (offset % 8);
    (bytes[byte_idx] >> bit_idx) & 1
}

// 設定位
fn set_bit(bytes: &mut Vec<u8>, offset: usize, value: u8) -> u8 {
```

### 2. BITCOUNT 範圍語義

```
BITCOUNT key                    # 整個字串
BITCOUNT key 0 0               # 第一個位元組
BITCOUNT key 0 -1              # 整個字串
BITCOUNT key 1 -1              # 從第二個位元組到末尾
```

### 3. BITPOS 語義

- `BITPOS key 0`: 找到第一個值為 0 的位
- `BITPOS key 1`: 找到第一個值為 1 的位
- 可選範圍參數

## 風險評估

1. **大 offset**: 超大 offset 可能導致大量記憶體分配
2. **效能**: BITCOUNT 在大字串上可能較慢
3. **NOT 運算**: BITOP NOT 只有一個鍵，不能多鍵
