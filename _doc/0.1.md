# kv5 規劃文件

## 版本資訊

- **版本**: 0.1.0
- **前置版本**: kv4 v0.1.0
- **專案狀態**: 進行中 (v0.1 已完成)

## 發展目標

kv5 是 kv4 的進階版，基於 kv4 v0.1.0 的功能進行擴展和優化。

### Redis 相容性目標

kv5 的 API 必須盡量和 Redis 完全相容，包括：
- 全部命令名稱、參數格式、回傳格式與 Redis 一致
- 錯誤訊息格式相容（ERR prefix）
- RESP 協議完全相容（RESP2）
- 可直接使用 redis-cli 操作
- 讓現有 Redis 客戶端（redis-py、jedis、go-redis 等）可以直接使用

## 功能規劃

### 第一階段：基礎強化（v0.1）

1. **Sorted Set 支援**
   - ZADD / ZRANGE / ZRANGEBYSCORE / ZRANK / ZSCORE
   - ZREVRANGE / ZREVRANGEBYSCORE
   - ZREM / ZCARD / ZCOUNT
   - ZINCRBY

2. **效能優化**
   - 連線池管理
   - 指令緩衝區優化
   - 記憶體使用優化

3. **測試強化**
   - 單元測試覆蓋率提升
   - 整合測試框架
   - test.sh 腳本

4. **程式碼重構**
   - 模組化拆分（store 超過 600 行）
   - 消除編譯警告
   - 加入 #![allow(dead_code, unused)]

### 第二階段：進階功能（v0.2）

5. **發布/訂閱（Pub/Sub）**
   - SUBSCRIBE / PUBLISH / PSUBSCRIBE
   - UNSUBSCRIBE / PUBSUB

6. **交易支援**
   - MULTI / EXEC / DISCARD
   - WATCH / UNWATCH

7. **效能監控**
   - INFO 命令增強
   - SLOWLOG
   - CLIENT LIST

### 第三階段：叢集準備（v0.3）

8. **哨兵模式基礎**
   - 主從複製（REPLICAOF / SLAVEOF）
   - ROLE 命令
   - 讀寫分離

9. **持久化增強**
   - RDB 格式優化
   - AOF 支援
   - 混合持久化

|------|------|------|
| ZADD | Sorted Set | 新增成員 |

| ZRANGEBYSCORE | Sorted Set | 按分數範圍取得 |
| ZRANK | Sorted Set | 取得排名 |
| ZSCORE | Sorted Set | 取得分數 |
| ZREM | Sorted Set | 移除成員 |
| ZCARD | Sorted Set | 成員數量 |
| ZCOUNT | Sorted Set | 分數範圍內數量 |
| ZINCRBY | Sorted Set | 分數遞增 |
| PUBLISH | Pub/Sub | 發布訊息 |
| SUBSCRIBE | Pub/Sub | 訂閱頻道 |
| PSUBSCRIBE | Pub/Sub | 訂閱模式 |
| UNSUBSCRIBE | Pub/Sub | 取消訂閱 |
| MULTI | Transaction | 開始交易 |
| EXEC | Transaction | 執行交易 |
| DISCARD | Transaction | 取消交易 |
| WATCH | Transaction | 監控鍵 |
| UNWATCH | Transaction | 取消監控 |
| REPLICAOF | Replication | 設定從節點 |

## 風險評估

1. **效能風險**: 新功能可能影響現有效能 → 需要效能測試
2. **相容性風險**: 新命令可能與未來 Redis 版本衝突 → 保持相容命名
3. **複雜度風險**: 交易和 Pub/Sub 增加複雜度 → 需要完善測試
